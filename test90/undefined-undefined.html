[CISCN2019 华东南赛区]Double Secret

今天又碰到了之前写过的有关debug模式未关闭而造成的源码泄露的题目，现在记录一下。

打开题目看见让我们找秘密。我们可以用目录扫描工具扫描出来一个secret页面。

让我们告诉秘密，尝试get方式传参secret。发现确实有回显。

随便输入测试后发现当我们输入字符超过一定位数时，便会跳转到debug界面。

这个模式我们之前已经说过它的危害了，最直接的就是他会将部分源码泄露，这样我们就能找到可趁之机来绕过一些过滤，从而拿到flag。

泄露的部分源码如下： 

if(secret==None):

    return 'Tell me your secret.I will encrypt it so others can\'t see'

rc=rc4_Modified.RC4("HereIsTreasure")   #解密

deS=rc.do_crypt(secret)



a=render_template_string(safe(deS))



if 'ciscn' in a.lower():

    return 'flag detected!'

我们已经知道这道题目时flask下的ssti注入题目，同时源码中也告诉我们我们我们输入的字符传入后是会被rc4算法解密，同时给出了密钥HereIsTreasure，我们可以不用专门了解这个算法，直接在网上找到rc4加密脚本，使用给出的密钥加密我们构造的ssti的payload即可。这样我们传入的payload被解密后就能成功ssti注入。

下面附上脚本 

import base64
from urllib.parse import quote
def rc4_main(key = "init_key", message = "init_message"):
# print("RC4加密主函数")
s_box = rc4_init_sbox(key)
crypt = str(rc4_excrypt(message, s_box))
return  crypt
def rc4_init_sbox(key):
s_box = list(range(256))  # 我这里没管秘钥小于256的情况，小于256不断重复填充即可
# print("原来的 s 盒：%s" % s_box)
j = 0
for i in range(256):
    j = (j + s_box[i] + ord(key[i % len(key)])) % 256
    s_box[i], s_box[j] = s_box[j], s_box[i]
# print("混乱后的 s 盒：%s"% s_box)
return s_box
def rc4_excrypt(plain, box):
# print("调用加密程序成功。")
res = []
i = j = 0
for s in plain:
    i = (i + 1) % 256
    j = (j + box[i]) % 256
    box[i], box[j] = box[j], box[i]
    t = (box[i] + box[j]) % 256
    k = box[t]
    res.append(chr(ord(s) ^ k))
# print("res用于加密字符串，加密后是：%res" %res)
cipher = "".join(res)
print("加密后的字符串是：%s" %quote(cipher))
#print("加密后的输出(经过编码):")
#print(str(base64.b64encode(cipher.encode('utf-8')), 'utf-8'))
return (str(base64.b64encode(cipher.encode('utf-8')), 'utf-8'))
